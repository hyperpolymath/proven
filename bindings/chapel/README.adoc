= Proven â€” Chapel Binding

FFI binding to *libproven*, a formally verified safety library implemented
in Idris 2 with a Zig FFI bridge exposing a stable C ABI.

*All computation is performed in the libproven core. This binding is a
thin wrapper; it does NOT reimplement any logic.*

Chapel is an HPC parallel computing language with native C interop, making
it natural to call libproven via `extern` declarations.

== Status: binding-tier-1 pilot (detachable harness)

This binding is the *binding-tier-1 standup pilot* for proven (see
xref:../../docs/adr/0001-binding-ci-template.adoc[ADR-0001] and
xref:../../docs/adr/0002-chapel-binding-standup.adoc[ADR-0002]). It runs
under a strict CI workflow that gates every PR touching the binding,
the C ABI header, or the Zig FFI layer.

[cols="1,3", options="header"]
|===
| Gate | What it proves

| `chapel-symbol-audit`
| Every WIRED extern in `symbol-manifest.txt` resolves against the built
  `libproven.so` (nm -D check).

| `chapel-build`
| Every Chapel module under `src/` passes `chpl --library --no-codegen`
  typecheck.

| `chapel-smoke`
| `smoke/hello_smoke` builds against the built `libproven.so` and exits 0.

| `chapel-tests`
| Every `tests/Test*.chpl` builds and runs all assertions to completion.

| `chapel-detachable-build`
| A sparse checkout of `bindings/chapel/` + `bindings/c/include/proven.h`
  + the workflow file is sufficient to rebuild + retest the binding.
|===

== Requirements

* Chapel 2.3.0 (`chpl --version`)
* `libproven.so` (Linux) available at `$PROVEN_LIB_PATH/libproven.so`
* `proven.h` (`bindings/c/include/proven.h`) reachable via
  `$PROVEN_INCLUDE_PATH` (default: `../c/include` relative to this
  binding)
* `just` (https://just.systems) for the build recipes

== Quick Start

[source,bash]
----
# 1. Set the libproven location (either local build or installed path).
export PROVEN_LIB_PATH="$(realpath ../../ffi/zig/zig-out/lib)"

# 2. Symbol-audit gate (5s; no compilation).
cd bindings/chapel
just check

# 3. Build the smoke target.
just build

# 4. Build + run all per-module tests.
just test
----

[source,chapel]
----
use Proven;

// WIRED: links + runs against libproven 0.9.0.
var trav = SafePath.hasTraversal("../../etc/passwd"); // some(true)
var crlf = SafeHeader.hasCrlf("v\r\nX-Inj: 1");       // some(true)

// Identify the linked libproven (debug-friendly).
writeln("libproven version: ", libraryVersion());

// GATED: declared per proven.h, NOT YET linkable.  Calling any of
// these will produce a linker error until proven#88 lands the
// corresponding Zig export.  See "Coverage matrix" below.
//   safeAdd / escapeHtml / isValidEmail / encode (URL) / ...
----

== Error Handling

All fallible operations return Chapel optional types (`T?`). Use pattern
matching to handle errors:

[source,chapel]
----
var result = SafePath.hasTraversal(userInput);
if result == none {
  writeln("FFI error reading input");
} else if result! {
  writeln("Path traversal detected â€” refusing");
} else {
  writeln("Path is safe");
}
----

== Coverage matrix (WIRED vs GATED)

WIRED â€” present in `libproven 0.9.0`, verified by `chapel-symbol-audit`,
exercised by `tests/`:

[cols="2,2,3", options="header"]
|===
| Chapel proc                       | C symbol                       | Test coverage
| `SafePath.hasTraversal(string)`   | `proven_path_has_traversal`    | `TestSafePath`, `TestFfiContract`
| `SafeHeader.hasCrlf(string)`      | `proven_header_has_crlf`       | `TestSafeHeader`, `TestFfiContract`
| `libraryVersion()`                | `proven_version`               | `TestLibraryInfo`, smoke
| `libraryBuildInfo()`              | `proven_build_info`            | `TestLibraryInfo`, smoke
| `provenFreeString(c_ptr)`         | `proven_free_string`           | (internal: called by `extractString`)
|===

GATED â€” declared in `src/LibProven.chpl` matching the documented C ABI
in `bindings/c/include/proven.h`, but not yet exported from
`libproven.so`. Tracked at
https://github.com/hyperpolymath/proven/issues/88[proven#88]. Calling
any of these from a Chapel program produces a linker error.

* SafeMath (`safeAdd`, `safeSub`, `safeMul`, `safeDiv`, `safeMod`,
  `safeAbs`, `safeClamp`, `safePow`, `safeNegate`)
* SafeString (`isValidUtf8`, `escapeSql`, `escapeHtml`, `escapeJs`)
* SafePath (`sanitizeFilename` â€” `hasTraversal` is WIRED above)
* SafeEmail (`isValid`)
* SafeUrl (`encode`, `decode`)
* SafeCrypto (`constantTimeEq`, `randomBytes`, `hexEncode`)
* SafeJson (`isValid`, `getType`)
* SafeDateTime (`parse`, `formatIso8601`, `isLeapYear`, `daysInMonth`)
* Lifecycle (`provenInit`, `provenDeinit`, `provenIsInitialized`) â€” note
  the ABI is in flux; the current Zig stub returns an opaque handle
  pointer, the documented ABI returns `int32_t`. Resolution lives with
  proven#88's ABI sign-offs.

== Build via the root workspace

The proven repo root exposes thin forwarders that build `libproven.so`
first, then delegate into this binding's harness:

[source,bash]
----
# From repo root.
just chapel-check    # symbol audit only
just chapel-build    # build smoke target
just chapel-test     # build + run all tests
----

Each of those is a one-liner that delegates to `just -d bindings/chapel
--justfile bindings/chapel/Justfile <subcommand>` with `PROVEN_LIB_PATH`
set to `ffi/zig/zig-out/lib`. They contain no Chapel-specific logic.

== Detachability

This binding is structured as a *self-contained package* so it can be
extracted to a standalone `proven-chapel` repo without surgery.

The full detachable set is:

[source]
----
bindings/chapel/
    Mason.toml                    # package metadata (chplVersion, license)
    Justfile                      # build + test recipes (uses PROVEN_LIB_PATH)
    README.adoc                   # this file
    symbol-manifest.txt           # WIRED symbol list
    scripts/symbol-audit.sh       # nm -D differ
    src/                          # Chapel sources (Proven.chpl + Safe*.chpl)
    smoke/hello_smoke.chpl        # end-to-end smoke
    tests/Test*.chpl              # per-module tests
bindings/c/include/proven.h       # C ABI header (source of truth)
.github/workflows/chapel-ci.yml   # CI workflow
----

To extract:

[source,bash]
----
# 1. Copy the detachable set into a fresh repo.
mkdir -p proven-chapel/{bindings,.github/workflows}
cp -r proven/bindings/chapel    proven-chapel/bindings/
cp -r proven/bindings/c         proven-chapel/bindings/
cp    proven/.github/workflows/chapel-ci.yml \
       proven-chapel/.github/workflows/

# 2. Confirm the harness still builds (libproven.so installed at $PROVEN_LIB_PATH).
cd proven-chapel/bindings/chapel
PROVEN_LIB_PATH=/opt/proven/lib just test
----

The `chapel-detachable-build` CI job in `chapel-ci.yml` runs this exact
sparse-checkout pattern on every PR â€” if extraction would regress, CI
goes red.
