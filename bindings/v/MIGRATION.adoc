= V Bindings — Deprecated (2026-04-12)
:toc:

== Status

These V-lang bindings for `libproven` are *deprecated* as of 2026-04-12
following the estate-wide V-lang ban (2026-04-10).

V-lang is banned across all hyperpolymath repositories.  Detection signal:
`v.mod` / `vpkg.json` (never `.v` — Verilog collision).

== Migration Targets

Use one of the canonical bindings instead:

=== Zig (preferred for FFI and systems code)

`proven/bindings/zig/` — comprehensive module-per-domain Zig bindings with
the same coverage as this file.  Idiomatic Zig with proper error handling.

[source,zig]
----
const proven = @import("proven");
const result = try proven.safe_math.safeDiv(42, 7);
----

=== Rust (preferred for application code)

`proven/bindings/rust/` — full Rust crate wrapping `libproven` via
`bindgen`-generated FFI.  Idiomatic Result<T, E> error handling.

[source,toml]
----
[dependencies]
proven = { path = "../../proven/bindings/rust" }
----

== File

`proven.v` (1 599 lines) — thin FFI wrapper around `libproven.so`.
All logic lives in Idris2; this file only marshals data.  The Zig and
Rust replacements provide identical coverage.

== Next Steps

. Remove `proven.v` from consuming projects.
. Replace imports with the appropriate Zig or Rust binding.
. The `v.mod` in this directory can be deleted once all consumers are migrated.
