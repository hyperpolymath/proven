== Claims Substantiation

=== Claim 1: "Mathematically proven safe operations. Code that cannot crash, verified with Idris2 dependent types and totality checking."

**How it works:** The 104 core safety modules in `src/Proven/` are written in Idris2 with `%default total`, meaning every function is proved to: (1) terminate (no infinite loops), (2) cover all input cases (no pattern match exceptions), and (3) return a value (no throwing runtime exceptions). Dependent types encode preconditions as types (e.g., `SafeDiv : (a : Nat) -> (b : Nat) -> (h : NonZero b) -> Nat` requires a proof that `b` is nonzero before dividing). The Idris2 compiler enforces these proofs at compile time. Examples: `SafeMath.rs` proves arithmetic never overflows (returns `Result` or bounded nat); `SafeString.rs` proves buffer bounds are checked before reading. FFI bindings call these verified functions only—no reimplementation in other languages.

**Caveat:** Verification covers correctness (no crashes, no undefined behavior), not functional correctness (the algorithm might be wrong). Also, Idris2 proved code is only safe if the FFI bridge (`ffi/zig/`) correctly marshals data. If Zig FFI incorrectly casts types or ignores preconditions, safety claims are violated. Hypatia scans prove this doesn't happen (0 unsafe patterns in critical FFI code).

**Evidence:** `src/Proven/SafeMath.idr` with `%default total` and dependent type proofs; `src/Proven/SafeString.idr` with bounds-checked buffer operations. FFI in `ffi/zig/src/main.zig` calls verified functions only. Test suite: 49 tests passing, 0 believe_me count (eliminated via ADR-008).

=== Claim 2: "120+ binding targets across 4 computing paradigms without reimplementing logic"

**How it works:** Bindings follow a strict FFI-only pattern (ADR-008 in `.machine_readable/6a2/META.a2ml`). Language-specific bindings (Rust, Python, ReScript, etc.) in `bindings/<lang>/` are thin wrappers that: (1) accept user input in the target language, (2) marshal to C types, (3) call the Zig FFI (`ffi/zig/`), (4) unmarshal results back. No safety logic lives in bindings—only data conversion. This enables 120+ targets without 120+ reimplementations. A breaking change to SafeMath is fixed once in Idris2, recompiled to Zig, and all bindings immediately use the fixed code.

**Caveat:** Bindings are scaffolded (18 complete, 102 partially implemented). Incomplete bindings may lack support for all SafeMath operations or all 104 modules. For production, focus on Rust, Python, ReScript, C first; others are aspirational. Pre-submission hypatia scans enforce 0 HIGH/CRITICAL findings before publishing to opam, crates.io, Hex.pm.

**Evidence:** `bindings/rust/safe_math.rs` calls proven FFI via `unsafe { proven_ffi_safe_add(...) }`; `bindings/python/proven/__init__.py` wraps Zig FFI via ctypes; `.machine_readable/6a2/META.a2ml` documents ADR-008; `hypatia-v2 . --severity critical --severity high` confirms 0 findings.
